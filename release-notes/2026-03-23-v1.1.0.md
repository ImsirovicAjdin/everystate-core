# @everystate/core v1.1.0

**Date:** 2026-03-23
**Previous:** v1.0.9

## Summary

Adds `store.derived(path, deps, fn)` to core. Derived paths auto-recompute when
their dependencies change. Manual `set()` on a derived path throws. Derivations
are cleaned up on `destroy()`.

This eliminates the manual `recomputeStats()` pattern, which was the most common
source of stale-UI bugs.

## API

```js
// store.derived(path, deps, fn) -> unsubscribe
const unsub = store.derived('stats.total', ['todos'], () => store.get('todos').length);

// Derived paths are read-only
store.set('stats.total', 99); // throws: "Cannot set derived path..."

// Remove a derivation (path becomes writable again)
unsub();
```

- **path** - the derived path (e.g. `'stats.total'`)
- **deps** - array of dependency paths (supports wildcards like `'todos.*'`)
- **fn** - computation function, returns the derived value
- **returns** - unsubscribe function that removes the derivation

## Before/After

### v1.0.9: Manual recomputation

```js
import { createEveryState } from '@everystate/core';

const store = createEveryState({
  todos: [
    { id: 1, text: 'Buy milk', done: false },
    { id: 2, text: 'Walk dog', done: true },
  ],
  stats: { total: 2, active: 1, done: 1 },
  filter: 'all',
});

function recomputeStats() {
  const todos = store.get('todos');
  store.batch(() => {
    store.set('stats.total', todos.length);
    store.set('stats.active', todos.filter(t => !t.done).length);
    store.set('stats.done', todos.filter(t => t.done).length);
  });
}

// Every handler must remember to call recomputeStats()
function addTodo(text) {
  const todos = store.get('todos');
  store.batch(() => {
    store.set('todos', [...todos, { id: Date.now(), text, done: false }]);
  });
  recomputeStats(); // forget this -> stale UI
}

function toggleTodo(id) {
  const todos = store.get('todos').map(t =>
    t.id === id ? { ...t, done: !t.done } : t
  );
  store.set('todos', todos);
  recomputeStats(); // forget this -> stale UI
}

function removeTodo(id) {
  store.set('todos', store.get('todos').filter(t => t.id !== id));
  recomputeStats(); // forget this -> stale UI
}
```

### v1.1.0: Derived paths

```js
import { createEveryState } from '@everystate/core';

const store = createEveryState({
  todos: [
    { id: 1, text: 'Buy milk', done: false },
    { id: 2, text: 'Walk dog', done: true },
  ],
  filter: 'all',
});

// Declare once. Never think about it again.
store.derived('stats.total',  ['todos'], () => store.get('todos').length);
store.derived('stats.active', ['todos'], () => store.get('todos').filter(t => !t.done).length);
store.derived('stats.done',   ['todos'], () => store.get('todos').filter(t => t.done).length);

// Handlers just mutate the source data
function addTodo(text) {
  const todos = store.get('todos');
  store.set('todos', [...todos, { id: Date.now(), text, done: false }]);
}

function toggleTodo(id) {
  store.set('todos', store.get('todos').map(t =>
    t.id === id ? { ...t, done: !t.done } : t
  ));
}

function removeTodo(id) {
  store.set('todos', store.get('todos').filter(t => t.id !== id));
}
```

## What changed

| | v1.0.9 | v1.1.0 |
|---|---|---|
| **Lines** | 34 | 22 |
| **Bug surface** | 3 call sites that can forget `recomputeStats()` | 0 - stats can't go stale |
| **`stats` in initial state** | Must be manually seeded with correct values | Not needed - computed on first `derived()` call |
| **`store.set('stats.total', x)`** | Allowed (can create inconsistency) | Throws - derived paths are read-only |
| **Adding a new handler** | Must remember to call `recomputeStats()` | Just mutate `todos`, stats update automatically |
| **`recomputeStats` function** | You write and maintain it | Doesn't exist |

The core difference: in v1.0.9, consistency is the developer's responsibility.
In v1.1.0, consistency is the system's responsibility.
